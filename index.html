C++ Primer Plus 学习笔记

发表于 2021-01-02 更新于 2021-01-20

第一章

  • Windows常用编译器：Cygwin, MinGW。
  • 先将源代码编译为目标代码文件，其后缀为.o，然后再链接其他程序，最后成为一个可运行程序a.out/a.exe。

第二章

  • 命名空间 using namespace std; 相当于将整个名称空间导入。
    可以只导入需要的部分，如：

  • endl 和 \n 的区别：
    endl会确保程序在继续运行前刷新输出，将内容立即显示在屏幕上。而’\n’并没有这样的机制。因此在某些情况下，可能会出现直到用户输入之后提示信息才显示的bug。

  • 一行代码中不可分割的元素叫做标记(token)，空格、制表符和回车统称为空白(white space)。

  • 连续使用赋值运算符，赋值将从右向左进行。

  • C++程序应当为程序中使用的每个函数提供原型

  • 关键字void指出函数不接受任何参数。如果省略void，让括号为空，则C++将其解释为一个不接受任何参数的隐式声明。

  • C++不允许将函数定义嵌套在另一个函数定义中

  • 命名空间的位置: “只让需要访问名称空间std的函数访问它是更好的选择”.

    • 将 using namespace std; 放在函数定义之前，让文件中的所有函数都能够使用std中的所有元素。
    • 将 using namespace std; 放在特定的函数定义中，让该函数能够使用std中的所有元素。
    • 在特定的函数中使用类似 using std::cout; 这样的编译指令，而不是 using namespace std;，让该函数能够使用指定的元素，如cout。
    • 完全不使用编译指令 using，而在需要使用std中的元素时，使用前缀 std::，如 std::cout<<"hello world"<<endl;。
